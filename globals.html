ZOptionSyntax

ZOptionSyntax: (this: void, args: keyof [string, string]) => Iterable<ZOptionInput>
  • Defined in run-z/optionz/src/option-syntax.ts:17

Supported command line syntax.

The syntax is a function that converts raw command line arguments to option input suitable to be recognized by option readers.

A syntax should be registered for options parser to respect it.

Type declaration

        An array of command line arguments to process.

      Returns Iterable<ZOptionInput>

      An iterable of option inputs. May be empty if the command line argument has another syntax.

longOptions

  • get longOptions(): ZOptionSyntax
    • Defined in run-z/optionz/src/option-syntax.ts:63

    Long command line option syntax.

    Supports the following option formats:

    • --name=VALUE
    • --name [VALUE [VALUE ...]]

    Option keys should be in --name format. Uses --*=* as a fallback option key for options -n --name=VALUE format. Uses --* as a generic fallback option key.

    Enabled by default.

    Returns ZOptionSyntax

shortOptions

  • get shortOptions(): ZOptionSyntax
    • Defined in run-z/optionz/src/option-syntax.ts:82

    Short command line option syntax.

    Supports the following option formats:

    • -name=VALUE. Corresponding key should be in -name format.
    • -name [VALUE [VALUE ...]]. Corresponding option key should be in -name format.
    • -n[m[o...]][VALUE]. Corresponding option key should be in -n* format.

    Uses -*=* as a fallback option key for options in -name=VALUE format. Uses -? as a fallback option key for one-letter options. Uses -* as a generic fallback option key.

    Enabled by default.

    Returns ZOptionSyntax
